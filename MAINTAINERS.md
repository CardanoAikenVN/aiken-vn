## Community Maintainers

Community contributors who have made significant contributions may be invited as maintainers.

To become a community maintainer:
1. Submit multiple quality pull requests
2. Participate in lesson reviews and feedback
3. Help other learners in community channels

## Maintainer Responsibilities

- Review pull requests within 3-5 business days
- Ensure content accuracy (especially Aiken code examples)
- Maintain Vietnamese language quality and diacritics
- Run `aiken check` on all smart contract changes
- Run `npm run build` to verify documentation site builds

## Decision Making

- Minor changes (typos, formatting): Any single maintainer can merge
- New lessons or content: Requires review from at least one core maintainer
- Architecture changes: Requires review from Lead Developer
- Smart contract changes: Must pass all existing tests (`aiken check`)
