# Codebase Summary

## High-Level Overview

Vietnamese Aiken is a Docusaurus 3.9.2 documentation site built with React 19 and TypeScript. The project combines static documentation content with a dynamic landing page featuring animations, accessibility features, and analytics tracking via Firebase.

### Technology Stack

| Layer | Technology | Version |
|-------|------------|---------|
| Framework | Docusaurus | 3.9.2 |
| UI Library | React | 19.0.0 |
| Language | TypeScript/JavaScript | 5.x |
| Styling | Tailwind CSS | 3.4.17 |
| Animations | Framer Motion | 12.23.25 |
| Icons | Lucide React | 0.555.0 |
| Backend | Firebase Firestore | 12.6.0 |
| Diagrams | Mermaid | (via @docusaurus/theme-mermaid) |
| Build Tool | Webpack (bundled with Docusaurus) | - |

## Development Workflow

### Local Development

```bash
# Install dependencies
npm install

# Start development server with hot reload
npm start
# Opens http://localhost:3000

# Build for production
npm run build

# Serve production build locally
npm run serve

# Clear Docusaurus cache (troubleshooting)
npm run clear
```

### Build Process

1. **Docusaurus Build**: Compiles Markdown to HTML, processes React components
2. **Tailwind CSS**: PostCSS plugin processes utility classes
3. **Asset Optimization**: Images and static files are optimized
4. **Output**: Static files generated in `build/` directory

### CI/CD Pipeline

The GitHub Actions workflow (`.github/workflows/ci.yml`) runs on:
- Push to `main` or `part-*` branches
- Pull requests to `main`

**Jobs:**
1. **Validate**: Read CLAUDE.md, verify contributor (jasong-03 only)
2. **Build**: Install dependencies, run build, upload artifact

### Content Development Workflow

1. **Create/Edit**: Write Markdown files in `docs/` directory
2. **Frontmatter**: Add required YAML frontmatter (title, sidebar_position, description)
3. **Preview**: Run `npm start` to preview changes
4. **Build**: Run `npm run build` to verify no broken links
5. **Commit**: Push changes to appropriate branch

## File Naming Conventions

### Documentation Files

```
NN_Descriptive_Title.md
```

| Component | Format | Example |
|-----------|--------|---------|
| Number | Two digits (01-99) | `01`, `02`, `13` |
| Separator | Underscore | `_` |
| Title | PascalCase or snake_case | `Installation`, `Control_flow` |
| Extension | Markdown | `.md` |

**Examples:**
- `01_Installation.md`
- `07_Custom_type.md`
- `12_Unit_test.md`

### Component Files

| Type | Convention | Example |
|------|------------|---------|
| React Component | PascalCase | `QuestTimeline.tsx` |
| TypeScript Types | lowercase | `types.ts` |
| Constants | lowercase | `constants.ts` |
| Utilities | camelCase | `sidebarToggle.js` |
| Styles | kebab-case | `custom.css` |

## Key Architectural Decisions

### 1. Docusaurus as Framework

**Rationale:** Optimized for documentation sites with built-in features:
- Markdown/MDX support
- Auto-generated sidebars
- Search functionality
- i18n support
- Static site generation for performance

### 2. Tailwind CSS Integration

**Rationale:** Utility-first approach enables:
- Rapid UI development
- Consistent design system
- Smaller CSS bundle with purging
- Easy dark mode support

### 3. Firebase for Analytics

**Rationale:** Firestore provides:
- Real-time data syncing
- Simple integration
- Free tier sufficient for current scale
- No backend server required

### 4. TypeScript for Landing Page

**Rationale:** Type safety for:
- Component props validation
- Better IDE support
- Reduced runtime errors
- Self-documenting code
